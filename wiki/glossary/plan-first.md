# Plan-First

一种 Agent 工作模式：先花时间理解需求、调研现有代码、制定方案，确认后再动手写代码。

## 解决什么问题

Agent 默认倾向于"马上开始写代码"。对简单任务这没问题，但对复杂任务，不规划就动手往往导致方向错误、返工、甚至破坏现有代码。Plan-First 强制 Agent 先输出一个计划（涉及哪些文件、怎么改、风险点），人类审核后再执行。

## 典型流程

1. **研究阶段** — Agent 读代码、读文档、理解现状
2. **规划阶段** — 输出结构化计划（要改什么、怎么改、预期结果）
3. **确认** — 人类审核计划，可能调整
4. **执行** — 按计划逐步实施

## 什么时候该用

- 跨多个文件的重构
- 不熟悉的代码库
- 高风险改动（数据库 schema、API 接口）

## 什么时候不必用

- 改个 typo、调个 CSS
- 你已经明确知道要怎么改，只需要 Agent 执行
