# Agent Loop

Agent 的核心运行循环：接收输入 → 思考 → 调用工具 → 观察结果 → 继续思考，直到任务完成或达到终止条件。

## 解决什么问题

普通 LLM 调用是单轮的：一个输入，一个输出。但真实任务需要多步操作——读文件、改代码、跑测试、看结果、再改。Agent Loop 让 LLM 能自主循环执行，而不是每一步都等人类指令。

## 基本流程

```
用户输入 → [决策: 下一步做什么]
              ↓
         [行动: 调用工具]
              ↓
         [观察: 读取工具结果]
              ↓
         [判断: 任务完成了吗?]
           ↙        ↘
         完成       继续循环
```

## 关键设计点

- **终止条件**：防止无限循环（最大 turn 数、超时、用户中断）
- **状态管理**：每一轮的上下文如何累积和压缩
- **错误恢复**：工具调用失败时怎么办

## 类比

开发者 debug 的过程：看报错 → 猜原因 → 改代码 → 跑测试 → 看结果 → 再改。Agent Loop 就是把这个人类行为模式自动化了。
