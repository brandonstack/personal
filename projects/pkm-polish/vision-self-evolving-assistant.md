# Kirby — 自我进化的 AI 个人助手

> 不是一个带 AI 的笔记 app，是一个帮你思考的伙伴，碰巧把你的想法存成了纯文本。

---

## Vision

一个有持久记忆、能自我进化、主动帮你思考和管理生活的 AI 伙伴。

它有自己的界面，但核心是一个**活的系统**——它观察你的行为，理解你的意图，替你做准备工作，在你需要之前就把东西准备好。

**一句话**：越用越懂你，而且你能看到它在成长。

---

## 为什么做这个

### 现在的 AI 产品都有同一个问题

**ChatGPT**：聊得好，但聊完就没了。没有持久记忆，不了解你。

**Notion AI / Mem.ai**：数据锁在平台，AI 只是被动的搜索/整理工具。

**Siri / Alexa**：用了十年还是那个样子，不会成长。

**Obsidian**：纯文本、不锁定、插件丰富——但没有 AI，全靠手动维护。

### 缺的是什么？

| | 现有 AI 产品 | Kirby |
|---|---|---|
| 交互模式 | 你问我答（被动） | 我观察、我思考、我主动 |
| 记忆 | 无 / 短期 | 持久 + 分层进化 |
| 数据所有权 | 锁在平台 | 纯文本，你拥有 |
| 成长性 | 静态不变 | 越用越懂你 |
| 信任 | 黑盒 | 透明（你能看到它的记忆和推理） |

**核心洞察**：门槛从"你要知道做什么、怎么做"变成了"你只需要活着，AI 帮你把碎片拼成图景"。

## Killer Features

### 1. 🔬 Digest（消化机制）— 核心差异化

从 Inquiry 项目继承并进化的核心功能。

**问题**：你每天看大量内容（文章、播客、对话），但 95% 看完就忘了。

**Kirby 的做法**：
- 你丢进来一篇文章 → Agent 自动提炼要点
- 给你 2-3 个苏格拉底式思考问题
- 你回答（哪怕一句话）→ 知识沉淀为你自己的
- 一周后 Agent 发现"你上周学的 X 和今天看的 Y 有关系" → 主动推给你

**没有人在做这件事。**

### 2. 🧬 自我进化（Self-Evolution）— 真正的差异化

不是一个静态工具，是一个**在成长的伙伴**。

#### 反思循环（Reflection Loop）

```
每天：
  - 回顾今天的对话，哪些回答好/不好？
  - 发现行为模式（你总是在 X 时间做 Y 事情）
  - 写 reflection 到 memory

每周：
  - 整理这周的 reflections
  - 更新自己的行为规则
  - 发现知识缺口（"我对 X 话题帮不上忙，需要学"）
  - 给用户一份简短的 "进化报告"

每月：
  - 自我评估（我变好了还是变差了？）
  - 清理过期的 memory
  - 提出改进建议
```

#### 记忆分层进化

```
Level 0: 记录事实    — "星在微软工作"
Level 1: 理解偏好    — "星喜欢 MVP 原则，不喜欢过度设计"
Level 2: 预测需求    — "星最近在准备转组，可能需要面试帮助"
Level 3: 主动建议    — "你上周提到的 X 和今天看的 Y 有关，要不要深入聊聊？"
```

#### 技能自生长

- 发现用户经常做某种操作 → 自动封装成 skill
- 发现某个领域帮不上忙 → 搜集资料、自学
- 发现 workflow 可以优化 → 重构自己的流程

#### 进化可见性

- 用户能看到 Agent 的记忆和规则变化（透明的 Git diff）
- "你上个月问我保险的问题我回答得不好，这个月我学了更多，要不要重新聊聊？"
- 这创造了**情感连接**——用户觉得"这是我的助手，它认识我"

### 3. 🗺️ Quest（学习路径）

不只是待办事项，是**知识领域的锚点**。

- 你对一个领域感兴趣 → 创建 Quest
- Agent 追踪你在这个领域的知识积累
- 自动发现覆盖度缺口："你了解了 A 和 B，但 C 是连接它们的关键，你还没接触过"
- 跨 Quest 发现："你学的保险规划和理财自由有交叉点"

### 4. 📥 Inbox（零摩擦输入）

任何东西都可以扔进来：
- 一个链接
- 一段想法
- 一张截图
- 一段语音
- 转发的聊天消息

Agent 负责整理、分类、digest。你的唯一工作是"扔进来"。

## 架构

### 核心原则

> **纯文本即 Ground Truth。Agent 读写文本。UI 只是视图。**

```
纯文本 (Git Repo) — 唯一数据源
  ├── App UI（浏览/交互）
  ├── Agent（自动化/对话/进化）
  ├── Obsidian（Power User 的编辑器）
  └── 未来任何工具（都只读写文本）
```

### Agent 内部结构

```
.kirby/
├── soul.md              # 自我认知 + 行为规则（可自我修改）
├── memory/              # 持久记忆（分层）
│   ├── facts.md         # Level 0: 事实
│   ├── preferences.md   # Level 1: 偏好
│   ├── predictions.md   # Level 2: 预测
│   └── daily/           # 每日记录
├── reflections/         # 反思日志
│   ├── weekly/
│   └── monthly/
├── skills/              # 自生成的技能
├── evolution-log.md     # 进化日志（用户可看）
└── config.md            # 运行配置
```

### Agent vs UI 分工

**Agent 做的（后台、异步、智能型）**：
- Feed 自动摄入（定时拉、去重、写 resources/pending）
- Digest 生成（总结、提问、关联发现）
- 对话沉淀（聊天中有价值的内容写回 repo）
- 主动推送（发现相关内容时通知）
- 反思 + 自我进化
- 定期 review（周/月回顾）

**UI 做的（浏览、选择、快速交互型）**：
- Inbox 浏览和分拣
- Digest 交互（回答问题、批注）
- Quest 查看和管理
- 对话界面
- 习惯追踪可视化
- Dashboard

## 目标用户

### Phase 1：开发者/极客（开源社区）
- 熟悉 Git、纯文本、CLI
- 能自己配置 Agent
- 对 plain text productivity 有热情
- 通道：Obsidian 社区、Hacker News、r/PKMS

### Phase 2：知识工作者
- 每天消费大量信息但难以沉淀
- 想学新东西但缺乏系统
- 需要好用的 UI，不愿碰终端
- 通道：独立 App

### Phase 3：学生
- 学习辅助 + 间隔重复 + 费曼检验
- Digest 机制天然适配学习场景
